Desafio Fullstack

Seu desafio será desenvolver um simples site agregador de notícias externas. Neste site existe uma única página com duas listagens de notícias.

As notícias deverão ser carregadas a partir de uma API. E a aplicação deverá ser desenvolvida como uma Single Page Aplication (SPA).

Envie instruções de como rodar seu projeto. Preferencialmente em um arquivo README. Envie também um dump do banco de dados utilizado.

API

  • Crie uma API consistindo de 2 endpoints.
    • Listagem de notícias
    • Computar like
  • A API deverá ser desenvolvida em PHP ou Node.js. Utilizando banco de dados MySQL ou MariaDB.

Listagem de notícias

  • No desktop a listagem de notícias deve possuir exatamente as mesmas posições como no Wireframe.
  • Caso a imagem da notícia que venha da API seja menor ou maior que o container, faça o mesmo respeitar o container. Caso seja maior, centralize a imagem dentro do container. Caso ele seja menor, faça com que o mesmo esteja centralizado horizontalmente e verticalmente, com background em cor sólida.
  • O card inteiro do item de notícia deverá ser clicável. Abrindo em outra aba o link da matéria.
  • Os cards da listagem principal deverão conter um botão de like (conforme wireframe). Ao ser clicado o botão deverá indicar que o voto foi computado para o usuário. Bem como realizar uma chamada para a API para incrementar a contagem de votos.
  • As informações que deverão conter no card são: imagem, título, descrição, quantidade de likes e data de publicação (conforme Wireframe).
  • A lista principal será ordenada por data de publicação e deverá conter no máximo 8 notícias por "página".
  • A lista secundária (sidebar) será ordenada por quantidade de likes e deverá conter no máximo 5 notícias.

Paginação

  • Carregar mais notícias conforme o usuário for scrollando a página ("Scroll infinito").
  • Observar que no mobile o scroll é em formato de slide. Enquanto em desktop o scroll é natural (vertical).

Modo Noturno

  • Ao clicar no botão switch de modo noturno a aplicação deverá trocar o esquema de cores de toda a UI para um tema escuro. As cores do Wireframe são apenas indicativas, use sua criatividade.
  • De mesmo modo, ao clicar novamente no botão, o esquema de cores deverá retornar ao tema claro.

Melhorias e sugestões

Que tal fazer umas melhorias no projeto? Seja deixando mais bonito, mais funcional ou mais otimizado.

Segue uma lista de sugestões do que você pode fazer:

  • Trabalhe em efeitos de hover nos itens e links.
  • Fique atento na semântica do HTML.
  • Tente criar transições e efeitos de carregamento.
  • Use animações quando onde e como achar melhor.
  • Use sua criatividade para utilizar fontes, cores, imagens decorativas, etc
  • Persista o estado do tema selecionado (Claro, escuro) pelo usuário mesmo após ele encerrar a sessão (Ex: locastorage).

O que estamos avaliando?

  • Qualidade do código
  • Capacidade de decisão do tipo de tecnologia utilizada
  • Conhecimento técnico
  • Velocidade de desenvolvimento
  • Performance da página. Tempo de carregamento, otimização, renderização, paint, etc.
  • Responsividade
  • Estruturação do banco de dados

Diferencial

  • Criar testes
  • Conhecimento em ⚛️ React (opcional)
  • Live demo. Faça deploy da sua aplicação em algum serviço gratuito.

Compatibilidade
